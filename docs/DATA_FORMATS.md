# Data Formats

This project uses two main file formats for motion data: **CSV** for raw motion input and **NPZ** for processed motion data used in training.

## 1. CSV Format (Raw Motion)

The CSV files (e.g., in `data/lafan_original/`) store motion frames as rows. Each row represents a single time step.

**Total Columns:** 36

| Column Index | Data Type | Description |
| :--- | :--- | :--- |
| **0 - 2** | `float` | **Base Position** (x, y, z) in world frame. |
| **3 - 6** | `float` | **Base Orientation** Quaternion (x, y, z, w). |
| **7 - 35** | `float` | **Joint Positions** (radians) for the 29 robot joints. |

### Joint Order (Indices 7-35)

## 2. NPZ Format (Processed Motion)

The `.npz` files are generated by `src/scripts/csv_to_npz.py` and contain NumPy arrays ready for the RL environment.

**Keys:**

*   **`fps`**: `(1,)` - Frame rate of the motion.
*   **`joint_pos`**: `(num_frames, 29)` - Joint positions in radians.
*   **`joint_vel`**: `(num_frames, 29)` - Joint velocities in radians/sec.
*   **`body_pos_w`**: `(num_frames, 3)` - Base position in world frame.
*   **`body_quat_w`**: `(num_frames, 4)` - Base orientation quaternion (w, x, y, z)*.
    *   *Note: Check specific implementation for quaternion order (w-first or x-first) as libraries vary.*
*   **`body_lin_vel_w`**: `(num_frames, 3)` - Base linear velocity.
*   **`body_ang_vel_w`**: `(num_frames, 3)` - Base angular velocity.
